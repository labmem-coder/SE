第2次作业的要求及说明

1.  必须完成的内容：本次作业要求完成下列用例对应的系统事件（标黄）的动态结构设计以及用例级别的静态结构的设计，除此之外要求组长完成在充电过程中出现一个充电桩故障的情况下，调度对象进行调度的交互过程：1）优先级调度；2）时间顺序调度；3）充电中故障恢复；

2.  可选完成的内容：1）单次调度总充电时长最短；2）批量调度总充电时长最短，这两个调度策略进行设计，要求交互图中能够清晰展示需求说明中的条件以及逻辑关系。每个选项的完成可以分别补充本次作业的5分，达到100分不再加分。

3.  本次作业由三个部分构成：其中1）系统架构的选择及说明占10%，2）动态结构设计占本次作业的80%，3）静态结构设计占10%。

4.  作业的最后必须说明系统事件的人员分配，这部分不给分，但如果没有最高扣5分；

5.  本次作业的动态结构设计要求使用UML的sequence
    diagram进行展示，静态结构设计要求使用UML class diagram
    进行表示（只需要使用依赖、定向关联以及继承关系），其中静态结构的类图中的类如果只有类的名称，就需要图后使用表格的方式对类的属性和方法进行说明（具体要求参看PPT）。

6.  动态结构和静态结构的设计图可以采用IBM
    RSA，Visio等绘图工具，手绘的方式完成。
